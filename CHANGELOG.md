# Changelog

All notable changes per release. Latest 5 detailed below; older versions: see
[GitHub Releases](https://github.com/ncmonx/icemage/releases). Each release ships
Linux + macOS (CI-built) and Windows binaries with SHA256 sidecars.

## v2.0.9

**Crash-hardening for path-argument commands on Windows Server.** A latent crash class: the throwing `std::filesystem::absolute` / `weakly_canonical` reach the PathCch API-set (`api-ms-win-core-path`), which fails to resolve on some Windows Server SKUs and raises `filesystem_error` 126 ("the specified module could not be found") — crashing any command that resolves a path argument (`context`, `graph`, `claudemd`, `plan`, `import`) on that host, even though `--version` and memory commands kept working. A new no-throw `core::absolutePath` helper (error-code overload + purely-lexical fallback, the same pattern as the v1.78 canonicalize fix) now backs eleven previously-unguarded sites, and `Config::projectDbPath` no longer calls the throwing `weakly_canonical`. Path-arg commands degrade to a lexical path instead of terminating. Full automated suite passes (1483 checks).

## v2.0.8

**Faster persona recall and leaner per-turn context.** Two token-efficiency improvements plus a test-harness fix. (1) **Ranked persona search** — `icmg profile search "<query>"` now uses an FTS5 / bm25 index over the persona store instead of a substring scan, so cross-project profile/skill/note recall returns the most relevant entries first; it degrades gracefully to a LIKE scan when FTS5 is unavailable, and the index self-syncs on every write (with a one-time backfill of pre-existing rows). (2) **Cross-turn injection dedup** — the per-prompt recall injector now skips a memory slice whose topic is a near-duplicate of one already surfaced earlier in the same session (a file-backed Jaccard window, reset on each new session), so the same context is not re-spent turn after turn — completing the 7-component injection governor. (3) **Fix:** the test entrypoint now sets `SetErrorMode(SEM_FAILCRITICALERRORS)`, so subprocess-spawning tests no longer raise a transient drive-not-found dialog on Windows. Full automated suite passes (1480 checks).

## v2.0.7

**Persona continuity, source provenance, and a cluster of correctness fixes.** Two identity-agnostic capabilities plus fixes surfaced during long-session use. **Features:** (1) **Persona-continuity scaffold** — `icmg persona init` seeds a set of neutral continuity zones in the per-install persona DB; a SessionStart hook surfaces a user-defined wake-up protocol and a Stop hook nudges a "refresh your notes" step on a meaningful turn, so an assistant's persisted context survives a fresh session or a context compaction. Nothing is hardcoded — every zone is a template the user fills. (2) **Source provenance** — `icmg store --source "<who>"` and `icmg profile add --source "<who>"` tag where information came from; `recall` and `profile get`/`list` display it as `[from: <source>]`. Free-text, defaults to `unknown`, and is metadata only — it never affects BM25 ranking. **Fixes:** (3) a corrupt global DB is now recovered cleanly on startup instead of crashing every global-scoped command; (4) the `persona` command no longer collides with the continuity `init` subcommand (one registration, both behaviors); (5) the drive-not-found popup dismisser now matches by the dialog's body text (heuristic 3), catching dialogs whose caption is empty rather than a bare `X:`; (6) a misplaced schema migration was moved from the global-DB array to the project-DB array, so a deployed binary initializes both DBs without error. Full automated suite passes (1437 checks).

## v2.0.6

**Long-session hardening round 2: a cleaner prompt history, a fixed internal-zone convention, and deterministic tests.** Continued marathon testing of v2.0.5 surfaced a tight cluster of correctness fixes. (1) `icmg prompt-capture` now strips harness/hook-injected noise (`<stop-hook-reminders>`, `<system-reminder>`, `[icmg]`/`MODE:` markers, slash-command echoes) before recording, so the auto-built history holds what you actually typed — not machinery (it had been recording a recurring "Stop hook feedback" cluster). (2) The internal `_`-zone convention was silently broken — `slugify` dropped the leading underscore, collapsing `_mode` to `mode` — so reuse could surface machinery (passphrase/mode/style) as a "similar prompt". `normalizeZone` now preserves a leading underscore, and `qa-suggest` / `qa-frequent` correctly exclude internal zones. (3) The mono test suite deletes stale `*_test.db` at startup, so back-to-back runs are deterministic (SQLite WAL/SHM sidecars had carried rows across runs, causing false failures). (4) `build.ps1`'s error scan matches only real MSVC `): error`/`LNK` lines, not runtime test-log output. (5) Every remaining `std::cin.rdbuf()` stdin reader (bench/compliance/compress/distill/expand/filter/json/memoir/shorten/tokens) is now isatty-guarded via `slurpStdinSafe`, completing the hook-hang hardening (#30704) — no unguarded stdin read remains. Full automated suite passes (1420 checks).

## v2.0.5

**Long-session hardening + a v2-roadmap feature: an honest, self-improving prompt loop.** A multi-hour stress session of v2.0.4 surfaced a cluster of fixes, plus one new capability from the v2 roadmap. **Fixes:** (1) `icmg suggest` now scores by command **name**, not just description-word overlap — "compress large output" maps to `compress`, not `expand` — and the gated `suggest --hook` gate is recalibrated so it fires only on a strong name-level match (it previously never fired; real scores were 0.1–0.25 under a 0.45 gate). (2) `icmg hook` (the Stop/precompact handler, spawned every turn) and `icmg shrink` (a PostToolUse hook) now read stdin via the isatty-guarded `slurpStdinSafe`, so a hook invoked without piped input returns immediately instead of risking a block (#30704). (3) `icmg flow` rejects a flow that needs an argument when none is given, instead of running a step with an empty token (no more junk `wflog add ""`). (4) `icmg prompt-capture` reads only the **tail** of the transcript, so its per-turn cost stays constant as a session grows to many MB. (5) `build.ps1`'s error scan is case-sensitive, so deliberate runtime test-log `ERROR` lines no longer show a false build error. **Feature:** `icmg profile qa-frequent [--count N] [--json]` clusters recurring prompts from the auto-recorded history (skipping internal `_` zones) and surfaces them as candidates to promote into a saved skill — closing the loop *capture → reuse → detect pattern → promote*. Full automated suite passes (1411 checks).

## v2.0.4

**Use what you built: command discovery, interlinked workflows, a persistent mode banner, and auto-recorded prompt reuse.** v2.0.0 added a lot of surface; v2.0.4 makes it findable and self-driving so capabilities stop going to waste. `icmg suggest "<intent>"` ranks the **live** command registry against a natural-language intent (model-free word-set scoring) so the long tail becomes discoverable instead of forgotten — and as a UserPromptSubmit hook (`suggest --hook`, gated) it auto-surfaces the most relevant command each turn. `icmg flow <name>` runs a named chain of **existing** commands in one shot (`change-done`, `sanity`, `refresh`; `{ARG}` substitution; `--dry-run`; fail-fast) so related features interlink. `icmg mode set/get/clear` keeps a cross-project session-mode banner in the persona DB and injects it every turn, so the agent stays oriented across compaction without you repeating context. `icmg profile qa-suggest "<prompt>"` reuses a past answer when a new prompt is similar (confidence-gated), and a new Stop hook (`icmg prompt-capture`) **auto-records** each turn's prompt→response into the per-day session zone — so reuse builds itself, no manual `qa-add`. Also: `icmg ingest --doc` rejects binary content in text-extension files. Full automated suite passes (1402 checks).

## v2.0.2

**Prompt-history CRUD, an active idle-compact advisor, and zone browsing.** Building on v2.0.0's prompt→response history, this round completes its lifecycle and activates a governor piece. `icmg profile qa-list` browses stored prompts (`--json` for scripts), `qa-forget` deletes one, and `profile zones` shows each zone with its prompt count — so the cross-project prompt corpus is navigable, not just searchable. The **idle-compact advisor (C5)** is now wired into the installed Stop hook: at the end of a turn, when context fill is high, it nudges you to `/compact` at a natural break instead of waiting for the mid-task wall (reads `icmg context-budget --percent`; opt out with `ICMG_NO_COMPACT_ADVISE=1`). Full automated suite passes (1373 checks).

## v2.0.1

**Long-session hardening: a hook can't hang the editor, and the init template bounds every hook.** Two fixes surfaced by a multi-hour stress session: (1) `icmg hookio` / `icmg correction` (via a shared `slurpStdinSafe`) now guard `std::cin` with an isatty check — a hook-spawned icmg invoked without piped input returns immediately instead of blocking forever on `cin.rdbuf()`, the root cause of process accumulation and hangs under sustained hook-spawn load; (2) `icmg init --install-hooks` now writes a `timeout` on every command hook in `settings.local.json`, so the editor-connection-timeout protection survives re-init/upgrade and ships to all users (previously a re-init silently reverted local timeout edits). Adds `icmg context-budget --percent`, which prints just the context-window fill percentage (override the window via `ICMG_CONTEXT_WINDOW`) — the signal the idle-compact advisor consumes. Full automated suite passes (1370 checks).

## v2.0.0

**Lean & Lossless Compaction governor + a zoned memory platform — icmg's flagship release.** The headline is a deterministic **context governor** (`icmg govern`) aimed at one pain: Claude Code's compaction stalling a session mid-task. Compaction is harness-locked and can't be made async, so the governor instead makes it **rarer, lossless, and un-surprising** with seven model-free pieces: budget-cap what icmg injects (C1); drop cross-turn duplicate slices via word-set Jaccard (C2); order context **U-shaped** so the most relevant sits at the extrema where models actually attend — mitigating the "lost in the middle" effect (C3); snapshot the working set before compaction and rebuild a hard-capped, pinned-only anchor after so nothing critical is lost (C4); an idle-compact advisor that nudges `/compact` at a natural break instead of the mid-task wall (C5, advisory — hooks can't trigger compaction yet); and structural trimming of noisy tool output (C6) and path-routed documents (`icmg ingest --doc`, C7) before they enter the window. Alongside it ships a **zoned profile/skill store** (`icmg profile add/get/list/search/forget`) in a cross-project persona DB, plus **prompt→response history** (`qa-add`/`qa-find`) so a repeated or similar prompt reuses its past solution instead of re-deriving it. Also: an IDE-hang fix — every hook is now bounded by a timeout so a stalled hook can't trip an editor's 60-second connection limit — and a new `working_set_snapshot` migration (0040). Full automated suite passes (1370 checks).

## v1.91.0

**Token-Efficiency v2: confidence-gated summaries, salience compression, and runtime dependency edges.** Three opt-in, local-first additions answering an external token-trimming critique: (1) **confidence-gated summarization** — `icmg gist` now scores each LLM summary (identifier-retention x length-sanity) and falls back to the deterministic heuristic when the summary is over-trimmed, off-topic, or hallucinated, so a lossy summary never reaches the main model; (2) **salience compression backend** — `icmg shrink --kind salience` keeps the most information-dense lines within a byte budget (pluggable scorer: heuristic now, llama-logprob perplexity later) as an LLMLingua-style alternative to the rule-based Tkil filters; (3) **runtime dependency edges** — `icmg graph runtime <trace>` parses python/node/gdb stack traces into `runtime_call` graph edges (dynamic execution flow, distinct from static `calls`). All additive; rule-based defaults unchanged. Full automated suite passes (1296 checks, +18).

## v1.90.0

**Security: the database key is now strictly hex-validated, and CodeQL stops scanning vendored code.** A CodeQL scan flagged two `cpp/sql-injection` paths where the SQLCipher encryption key (from an env var or key file) is interpolated into a `PRAGMA key=x'…'` blob literal — and `PRAGMA` cannot use bound parameters. v1.90 adds a fail-closed `isHexKey` gate inside `resolveDbKey`: any key that isn't strict hex (e.g. one containing a quote) is rejected and resolves to no key, so a crafted value can't escape the literal. It also adds a CodeQL config that path-ignores `third_party/` (vendored llama.cpp / sqlcipher), so scanning focuses on first-party code and ~159 upstream-owned findings are cleared. Full automated suite passes (1278 checks, +5).

## v1.89.0

**Graph prune, zone-scoped memory consolidation, multimodal graph nodes, and multi-agent work leases.** Four additions: (1) `icmg graph prune [--dry-run]` removes graph nodes whose backing file no longer exists, clearing scan-pollution (dead temp/build artifacts) that bloats the default zone; (2) `icmg memory consolidate --zone <name>` scopes near-duplicate collapse to a single zone, and the over-count hint now points to the correct command; (3) ingested media (`icmg ingest` image/PDF OCR) is recorded as a first-class graph node (`kind=multimodal`) so `icmg context`, graph queries, and zones surface it like source files; (4) `icmg session claim --scope <s>` / `leases` / `release` give conflict-free work leases across concurrent agents sharing one DB (stable identity via `ICMG_AGENT_ID`, stale leases auto-reclaimable). Full automated suite passes (1273 checks, +17).

## v1.88.0

**First CI-built release: Linux and macOS binaries are built and published automatically by GitHub Actions.** Pushing a version tag triggers a multi-platform matrix that compiles the full-feature build (ONNX + tree-sitter + llama/Vulkan + SQLCipher) on Linux and macOS, packages `tar.gz` + SHA256, and publishes the GitHub release — no manual cross-compilation. Windows is built locally (MSVC + sccache) and uploaded alongside. Adds release-automation scripts: `bump-version.ps1` syncs the version across all three source-of-truth files (`version.hpp` + `icmg.rc` + `CMakeLists`), and `release-win.ps1` refuses to upload unless the built binary's `--version` matches the tag. Full automated suite passes (1256 checks).

## v1.87.0

**Fix: `icmg update` and `icmg fetch` no longer fail on plain PowerShell**. Since v1.81, running icmg from a normal PowerShell prompt (not MSYS/Git-Bash) broke every command that calls the GitHub API or fetches a URL — `icmg update --check/--apply`, `icmg fetch`, the self-upgrade check — all returned "failed to query github (network or rate-limit)" even on a healthy connection. Two compounding causes: (1) in PowerShell, `curl` is an alias for `Invoke-WebRequest`, not the real curl, so its output was never the raw response body; (2) the internal shell wrapper passed commands to `pwsh -Command "..."` without escaping inner double-quotes, so a `-H "User-Agent: ..."` header silently broke argument parsing. v1.87 routes through `curl.exe` explicitly and escapes quotes in the wrapper — fixing all HTTP/JSON callers at once. Full automated suite passes (1256 checks).
