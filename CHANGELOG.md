# Changelog

All notable changes per release. Latest 5 detailed below; older versions: see
[GitHub Releases](https://github.com/ncmonx/icemage/releases). Each release ships
Linux + macOS (CI-built) and Windows binaries with SHA256 sidecars.

## v2.1.0

**One-turn code search, a visible context budget, one-line install, and 6 more languages.** (1) **`icmg find "<intent>"`** -- one-shot multi-file semantic search, IDF-ranked (an exact identifier beats common words), returns just the relevant line windows from the right files, collapsing a Read->Grep->Read chain into one turn. (2) **`icmg statusline`** (auto-wired by `icmg init`) puts the per-model-honest context budget in Claude Code's status bar every turn -- the invisible token cost made visible. (3) **`icmg bench savings`** measures token reduction on YOUR repo, reproducibly (~46% fewer tokens to read a ~1000-file tree via `icmg context` vs raw reads). (4) **One-line installers** (`scripts/install.sh` / `install.ps1`) fetch, checksum-verify, and install the latest release -- no build step. (5) **Six new graph languages** -- Ruby, Swift, Kotlin (now dedicated; was mis-mapped to Java), Scala, Lua, Dart -- bringing first-class import/symbol extraction to 14 languages. (6) Cleanup: a stale Python PreCompact hook that spammed `python3: command not found` on Python-less servers is now dropped by `icmg init`. 1673/1673 ctest (Windows + Linux).

## v2.0.15

**Token-efficiency leap, per-model honest budgets, and the Windows Server 2019 crypto crash fixed at the root.** (1) **`icmg context <file> --for "<intent>"`** returns only the lines relevant to an intent -- each line scored against the intent terms, windows built around hits, merged, top windows emitted with line numbers -- ~30 relevant lines instead of ~500, no `--lines A-B` guessing. (2) **Read-dedup stub** -- re-reading an unchanged file (a context cache hit; the key already includes mtime+size, so any edit misses) emits a compact "already shown this session" stub instead of re-dumping the body (~97% saved on the re-read; `--full` / `ICMG_NO_DEDUP_STUB` re-emits in full). (3) **Pre-exec command densifier** in `icmg run` rewrites noisy commands to emit less before the output filter runs (`git status -> --porcelain=v2 --branch`, `pytest -> -q --tb=line`, `tsc -> --pretty false`, ...), pure and guarded against shell composition and flags you set yourself. (4) **Per-model honest meters** -- a model context-window registry (opus-4/gemini 1M, gpt-4o/deepseek 128K, default a safe 200K) replaces a hardcoded 1M that lied on smaller windows, and a pricing registry replaces a hardcoded Sonnet rate that understated Opus cost ~5x. (5) **Server 2019 err126 root fix** -- SQLCipher's encrypted writes drew randomness through a legacy CryptoAPI module absent on headless Server Core; `icmg` now routes OpenSSL's RNG onto `BCryptGenRandom` (always present) so `context`/`graph update` run fully instead of degrading. 1657/1657 ctest (Windows + Linux).

## v2.0.14

**Windows-Server crypto-crash resilience, self-diagnosing module errors, and a command "you-are-here" map.** On hosts where a runtime crypto module is absent (a Windows Server SKU where SQLCipher's write-side crypto lazily loads a DLL that isn't there -> `err126 "specified module could not be found"`), `icmg context` no longer crashes — it degrades to emitting the raw file so the caller keeps read access, and the graph scanner skips an un-writable node instead of crashing or hanging. Errors now diagnose themselves: an `err126` crash prints the last successfully-loaded DLL plus an actionable hint, `ICMG_TRACE_DLL=1` streams the full load order, and a new `icmg doctor --deps` walks each bundled DLL's PE import table (static **and** delay) to name any module that won't resolve on this machine — no Process Monitor or admin required. New navigation: `icmg map <cmd>` shows a command's related neighbors (derived from the live registry, so it never rots), every command's `--help` ends with a `related:` footer, and `icmg doctor` flags accidental near-duplicate commands. Housekeeping: the rich diagnose-and-auto-fix `doctor` is now canonical (a duplicate registration had been shadowing it), the quick DB/schema probe moved to `icmg db-check`, and doctor's bundled-DLL check matches the real shipped MSVC set. Full automated suite passes (1633 checks).
## v2.0.13

**Audit-grade read-only enforcement, a no-bypass escape hatch, and a service-process bloat fix.** Two enforcement additions and one infrastructure fix. (1) **`icmg strict audit`** is a read-only deny mode for token-frugal audits: native full-file `Read` is hard-denied and redirected to `icmg context` (~80% fewer tokens) while targeted slices (offset/limit) stay allowed, and browser-automation MCP tools (puppeteer/playwright) are denied at PreToolUse because their DOM/screenshot payloads bypass the command filter entirely. It is safe precisely because an audit never edits — hard-denying `Read` globally would break Claude Code''s Read-before-Edit contract, so the mode is gated behind a flag, never the default. (2) **`RAW=1` no longer bypasses pure read/search verbs** (`cat`/`grep`/`sed`/`head`/`tail`/`find`/…): these always have an icmg equivalent, so the bypass is denied, with an escalating hourly cooldown cap that also denies legitimate-but-spammed `RAW=1` use until it ages out — stopping the escape hatch from becoming a habitual token leak. (3) **Fix:** the `icmg service` singleton guard created a `Global\` named mutex, which requires `SeCreateGlobalPrivilege`; on a normal non-elevated session `CreateMutexA` failed with `ERROR_ACCESS_DENIED` and the guard fell open, so the per-prompt autostart hook spawned dozens of duplicate background services (DB-lock contention that hung `graph update`, a dead-drive "insert disk" popup storm, and named-pipe collisions). The guard now uses the `Local\` namespace, which holds without elevation. A latent companion bug — the per-event hook injection cache replaying a stale per-file decision within its 30s window — was also fixed (input-dependent hook events bypass the cache). Full automated suite passes (1600 checks).
## v2.0.12

**Frictionless capture, unread-aware messaging, one-shot session resume, and zero-LLM event memory.** Four small high-leverage tools. (1) **`icmg store --quick "<msg>"`** drops a note with no topic — an auto `quick:<epoch>` is assigned — for friction-free capture mid-task. (2) **`icmg msg check`** peeks unread cross-session messages since a persisted, auto-advancing cursor, so you never pass `--since` by hand. (3) **`icmg wake-up --resume`** appends persona identity and recent moments to the session briefing, re-hydrating continuity in one command. (4) **`icmg auto-extract`** is a Layer-0, zero-LLM memory capture for tool events: a successful `git commit` becomes a workflow-log note, a failing build's error becomes a known-issue candidate, and everything else is skipped silently — enriched with rule-based entity extraction (URLs, IPs, env vars, @mentions) so a captured event is searchable by what it referenced, all without a model call. Full automated suite passes (1593 checks).
