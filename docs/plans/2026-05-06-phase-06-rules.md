# Phase 06: Per-folder Rules + Inheritance

> **For Claude:** REQUIRED SUB-SKILL: Use `executing-plans` or `subagent-driven-development` to implement this plan.

**Goal:** Sistem rules per-folder dengan hierarki inheritance — rule parent berlaku ke child kecuali di-override oleh rule lebih spesifik.
**Architecture:** RuleStore CRUD di SQLite. RuleResolver load semua rules yang prefix-match ke path, sort by specificity, merge.
**Tech Stack:** C++17, SQLite
**Assumptions:** Phase 01 selesai.

```cpp
struct Rule {
    int64_t id = 0;
    std::string scope_path;   // "/" = root, "src/" = src subtree
    std::string rule_type;    // coding|arch|workflow|model|custom
    std::string name;
    std::string content;
    int priority = 0;
    bool active = true;
    int64_t created_at = 0;
};

class RuleStore {
public:
    explicit RuleStore(core::Db& db);
    int64_t add(const Rule& rule);
    bool remove(int64_t id);
    bool setActive(int64_t id, bool active);
    std::vector<Rule> all() const;
    std::vector<Rule> forPath(const std::string& path) const;
    // Returns rules sorted: root first, most-specific last
};
```

### Task 2: Rule resolver (inheritance logic)

**Files:**
- Create: `src/rules/rule_resolver.hpp`
- Create: `src/rules/rule_resolver.cpp`

```cpp
class RuleResolver {
public:
    explicit RuleResolver(RuleStore& store);

    // Get all applicable rules for path, ordered by specificity
    // Rule lebih specific (longer scope_path) menang atas rule yang lebih general
    std::vector<Rule> resolve(const std::string& file_path) const;

    // Check apakah ada konflik (2 rules di level sama dengan name sama)
    std::vector<std::pair<Rule,Rule>> conflicts(const std::string& file_path) const;
};
```

**Inheritance logic:**
- scope_path "/" → berlaku ke semua file
- scope_path "src/" → berlaku ke semua file di src/ dan subdirektori
- scope_path "src/api/" → berlaku ke src/api/ dan subdirektori
- Matching: `file_path.starts_with(scope_path)`
- Sort by: `scope_path.length()` ascending → root rules dulu, specific rules terakhir
- Conflict: sama name, sama type, di scope_path sama → report conflict

## Amendments from Architecture Review

### HIGH Fixes

**A1 — UNIQUE constraint pada rules**
Tambahkan ke schema migration:
```sql
-- Prevent duplicate rules di scope yang sama
CREATE UNIQUE INDEX idx_rules_unique ON rules(scope_path, rule_type, name)
WHERE active = 1;
```
`RuleStore::add()` throw `RuleConflictError` jika UNIQUE violated.
`icmg rule add ... --update` untuk update existing rule.

**A2 — Conflict detection**
`RuleResolver::conflicts()` tambahkan ke output `icmg rule apply`:
```
⚠️  Conflict at scope src/api/:
  #7  arch: input-validation (priority 0) — "Validasi semua input"
  #12 arch: input-validation (priority 0) — "Input boleh nullable" ← konflik!
  Resolution: last-added wins (id=12). Use --priority to override.
```

**A3 — Trie untuk rule lookup (performance)**
Jika rules > 100, prefix-tree lookup lebih cepat dari linear scan:
```cpp
class ScopeTrieNode {
    std::unordered_map<char, std::unique_ptr<ScopeTrieNode>> children;
    std::vector<Rule> rules;
};
```
Build trie sekali saat startup, invalidate saat rule add/remove.
