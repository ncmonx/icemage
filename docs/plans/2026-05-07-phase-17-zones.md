# Phase 17 ŌĆö Zone Partitioning

**Goal:** Partition memory + graph nodes into named zones (subsystem, layer, bounded-context) for scoped recall, sharper BM25 IDF, and per-zone visualization.

**Why:** With 200+ nodes a flat namespace dilutes BM25 scoring. Zones cut recall scope 5-10├Ś and let users target one subsystem (e.g., `--zone sync`) without cross-talk.

**Tech:** SQLite migration, regex-based pathŌåÆzone mapping, CLI flag plumbing.

**Estimate:** 1-2 days.

**Assumptions:**
- A node belongs to exactly one zone (no overlap). Fallback zone `default`.
- Auto-detection by path prefix is good enough for 90% of projects; manual `icmg zone assign` covers the rest.
- Zone column is nullable ŌĆö existing rows treated as `default` until rescan.
